# 리액트에서 index를 key값으로 사용하면 안되는 이유에 대해서 설명해주세요.

## 안되는 이유

- 배열의 요소들이 추가되거나 삭제될 때, 배열의 순서가 바뀌는 경우 문제가 발생할 수 있기 때문입니다.
- 리액트는 key를 통해 어떤 요소가 변경, 추가, 삭제되었는지 추적합니다. 그러나 index를 key로 사용하면 리액트가 배열의 요소들을 잘못 인식할 수 있습니다.
- 배열에 새로운 요소가 추가되면 그 뒤에 모든 요소의 인덱스가 바뀌어 불필요한 리렌더링이나 상태를 잘못 처리할 수 있습니다.
- 이로 인해 성능 문제가 발생하거나, 사용자 입력 상태 같은 요소가 의도치 않게 초기화되는 등 버그가 생길 수 있습니다.
- 정적인 리스트에선 index를 사용해도 무방합니다.

## 그럼 무엇을..?

- 데이터의 유일성을 보장하고 변하지 않는 값을 사용하는 것이 좋습니다.
- 데이터베이스에서 내려오는 고유ID 값을 사용하는 것이 좋습니다.
- key로 고유한 값을 사용하기 위해 `"${title}_${name}"` 등 여러 필드를 합칠 수 있습니다.
- 렌더링 이전 시점에서 UUID 혹은 랜덤 값을 생성하여 사용할 수 있습니다.
